How It Works

Here's the code behind the demo. First, the main heading is defined by _data/resume.yml. You can specify the theme color here too.

photo: "http://url/to/photo.jpg"
name: "Elle Kasai"
copy: "UI/UX Designer from Japan..."
color: default

Next, the contact info section is defined by _includes/nav.html. You can use Font Awesome icons.

<p>
  <i class="fa fa-twitter fa-lg fa-fw"></i>
  <a href="https://twitter.com/ellekasai/">@ellekasai</a>
</p>

...

Finally, the cards are defined by markdown files under the _posts directory. It uses YAML front matter.

These files are put together by Jekyll, which is supported by GitHub Pages.

Also, card heights are fixed by CSS. So, if all the texts fit within each card when viewed on the web, they'd also fit within each card when clicking the "Saved as PDF" button.

The Story Behind ResumeCards

I've been looking for a UI design job in Vancouver, Canada, and one recruiter had asked me to submit a PDF resume. I wasn't sure why it had to be in PDF though, since anyone can learn about my professional experience on my about page, LinkedIn, and GitHub.

There are services that convert your LinkedIn or GitHub profile to a PDF file, but I couldn't find a design template I like. Also, my LinkedIn profile was too long to fit in one page, and my GitHub profile was too short. Rewriting the text was necessary.

Since I had to rewrite the text anyway, I decided to see if I can use Markdown and Jekyll to build a nice resume that looks great both on the web and in PDF. After some brainstorming, I realized that card UIs would work great. They're responsive design friendly. And by specifying the height manually, they can be made to fit in one page.

I decided to open source this, and made it so that users can pick the color they like. This is how ResumeCards was born.
